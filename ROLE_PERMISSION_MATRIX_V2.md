# Role Permission Matrix — School Connect v2

## Admin
Admin should be able to read/write/manage these operational modules:
- students
- staff
- parents
- classes
- subjects
- results
- attendance
- report cards
- academic records
- timetable
- CBT
- announcements
- events
- gallery
- library / digital library
- assignments
- parent-child mapping
- directory
- departments
- result broadcast
- complaints
- leave
- visitors
- hostel
- transport
- alumni administration
- promotion
- finance
- fees
- inventory
- certificates
- lesson plans / scheme of work oversight

## Staff / Teacher
Staff should be able to read/write operational academic modules:
- attendance
- results
- CBT manager
- report cards
- broadsheets / academic records
- lesson plans
- scheme of work
- timetable
- announcements
- inbox / messages where enabled
- complaints handling where enabled
- certificates
- assignments
- e-resources

## Parent
Parent should be able to read child-linked pages:
- child dashboard / student profile
- fees & payment history
- results
- assignments
- messages / inbox where enabled
- complaints
- report cards
- attendance
- timetable
- announcements

## Student
Student should be able to read or use self-facing pages:
- assignments
- timetable
- e-resources
- results
- profile
- certificates
- complaints
- inbox
- announcements
- report cards
- CBT exam page

## Enforcement rule
Permissions must be enforced at:
1. navigation
2. page entry
3. form/action level
4. database/RLS level
